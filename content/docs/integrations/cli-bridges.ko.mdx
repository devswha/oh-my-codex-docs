CLI bridge는 OMX 세션 바깥의 다른 AI CLI나 모델에 질문을 위임하는 경량 명령입니다. `$ask-claude`, `$ask-gemini`, `$ccg` 세 가지가 가장 흔한 cross-CLI 패턴을 담당합니다.

## 무엇인가

각 bridge는 대상 CLI binary를 fresh subprocess로 실행하고, 질문이나 작업을 보내고, 응답을 현재 OMX 세션으로 돌려받습니다. subprocess는 현재 컨텍스트를 상속하지 않고 새로 시작하기 때문에, bridge 호출은 active working context와 격리된 clean-room query가 됩니다. 현재 컨텍스트를 오염시키지 않으면서 외부 시각이 필요할 때 유용한 이유입니다.

## 세 가지 bridge

- **`$ask-claude`** — Claude CLI subprocess에 질문을 그대로 보냅니다. 빠른 조회, 요약, Codex 컨텍스트를 건드리지 않고 Claude의 추론을 가져오고 싶을 때 사용합니다.
- **`$ask-gemini`** — Gemini CLI subprocess를 실행합니다. 대용량 컨텍스트 조회나 Google 모델의 관점을 비교할 때 특히 유용합니다.
- **`$ccg`** — Codex, Claude, Gemini 세 모델에 병렬 fan-out 후 응답을 하나로 합성하는 tri-model 워크플로우입니다. `$ccg` skill 내부에서 사용되며, 특정 접근법을 고르기 전에 넓은 coverage가 필요할 때 씁니다.

## 예시

단일 모델 자문:

```
codex
> $ask-claude "what is the recommended retry strategy for idempotent HTTP endpoints?"
```

```
codex
> $ask-gemini "summarize the tradeoffs between RSA and ECDSA for JWT signing"
```

tri-model consensus:

```
codex
> $ccg "design a rate-limiting strategy for a public REST API with 10k RPS"
```

skill layer를 거치지 않고 shell에서 직접 호출하는 방법:

```bash
# Ask Claude directly from a shell script
echo "explain this error: $(cat error.log)" | claude --print
```
