核心钩子是运行 `omx setup` 时 OMX 安装的钩子处理程序。它们无需任何额外配置，即可将 OMX 的工作流技能与 Codex 生命周期事件连接起来。每个钩子只负责一项职责，当其关注点处于非活动状态时不会干预。

## 概述

OMX 提供四个注册在 `.codex/hooks.json` 中的内置钩子。所有四个都通过 `dist/scripts/codex-native-hook.js` 路由，这个单一入口点根据 Codex 事件类型分发到正确的处理程序。配置位于 `.codex/config.toml` 的 `[features] codex_hooks = true`。`.omx/hooks/*.mjs` 中的插件钩子扩展了这个接口，但不会替换它。

## 内置钩子

- **keyword-detector**（`UserPromptSubmit`）— 扫描每个用户提示中的魔法关键词短语。匹配时，将技能激活状态写入 `.omx/state/`，并可在前面添加开发者上下文。支持 `$skill` 显式调用和对"team"、"parallel"等模糊词汇带有意图守卫的隐式短语匹配。参见[魔法关键词](/zh/docs/hooks/magic-keywords)。
- **ralph stop-gate**（`Stop`）— 当模型停止时检查 Ralph、autopilot、ultrawork 或 ultraqa 运行是否仍处于活动状态。若活动且尚未达到终止状态，则返回带有继续原因的 `decision: "block"` 以保持运行。从 `.omx/state/{mode}-state.json` 读取状态。
- **pre-tool-use guard**（`PreToolUse`，仅 Bash）— 对破坏性命令（`rm -rf dist`）发出警告，并阻止缺少必要 Lore 格式尾注（`Co-authored-by: OmX <omx@oh-my-codex.dev>`）的内联 `git commit` 调用。
- **post-tool-use advisor**（`PostToolUse`，仅 Bash）— 解析非零退出码、权限拒绝错误和路径未找到消息，然后将纠正指导注入下一个提示上下文。

## 示例

验证 OMX 钩子已安装且配置正确：

```bash
omx doctor
```

关键词触发后检查原始钩子状态：

```bash
cat .omx/state/skill-active-state.json
```
