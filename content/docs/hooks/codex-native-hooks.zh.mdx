Codex CLI 暴露了一个原生钩子接口：当 `.codex/config.toml` 中设置 `[features] codex_hooks = true` 时，Codex 通过读取 `.codex/hooks.json` 中的注册信息，在定义的生命周期点调用外部处理程序。OMX 直接使用此接口，而不是用 shell 脚本包装 Codex 二进制文件。

## 概述

`omx setup` 写入两个文件：`.codex/config.toml`（启用 `codex_hooks`）和 `.codex/hooks.json`（注册 OMX 处理程序命令 `dist/scripts/codex-native-hook.js`）。安装过程会保留 `.codex/hooks.json` 中已有的用户管理的钩子条目。`omx uninstall` 只删除 OMX 管理的条目；用户钩子保持不变。替代方案——在 Codex 二进制文件周围包装 shell——仍通过 `omx tmux-hook` 支持，但不是原生事件的主要路径。

## 原生与运行时回退覆盖

| OMX 关注点 | Codex 事件 | 状态 |
|---|---|---|
| 会话记账、wiki 启动上下文 | `SessionStart` | 原生 |
| 关键词检测、技能状态播种 | `UserPromptSubmit` | 原生 |
| Bash 安全检查、提交格式守卫 | `PreToolUse`（仅 Bash） | 原生-部分 |
| 命令未找到指导、退出码审查 | `PostToolUse`（仅 Bash） | 原生-部分 |
| Ralph / autopilot / ultrawork / ultraqa 继续 | `Stop` | 原生-部分 |
| 停滞轮次自动推进 | `Stop` | 原生-部分 |
| 会话结束 wiki 捕获、`session-idle` | — | 运行时-回退 |
| 非 Bash 工具拦截 | — | 运行时-回退 |

"原生-部分"表示原生路径处理常见情况；边缘情况回退到 notify-hook 或 tmux 运行时路径。

## 示例

验证原生钩子已安装且配置正确：

```bash
omx doctor
```

运行冒烟测试以确认原生钩子路径端到端执行：

```bash
omx exec --skip-git-repo-check -C . "Reply with exactly OMX-EXEC-OK"
```

插件分发日志写入 `.omx/logs/hooks-YYYY-MM-DD.jsonl`。
