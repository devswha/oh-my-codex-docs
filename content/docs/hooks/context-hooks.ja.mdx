---

コンテキストフックは、構造化された知識で次のプロンプトを充実させる OMX フックのサブセットです。ユーザーメッセージを変更する代わりに、Codex が注入されたコンテキストとして読み取る `<system-reminder>` タグを発行します。これにより、ユーザーメッセージをクリーンに保ちながら、モデルが常に最新のプロジェクトメモリ、アクティブなディレクティブ、notepad エントリを確認できるようにします。

## 概要

コンテキストフックが発火すると、1 つ以上の知識ソース——`.omx/state/`、`.omx/wiki/` の wiki、プロジェクトメモリ、または notepad——を読み取り、関連するコンテキストをプロンプトコンテキストに追加される `<system-reminder>` ブロックとしてフォーマットします。モデルはこのコンテキストをユーザー指示ではなく権威ある背景情報として認識するため、すべてのターンに影響を与えるべき永続的な事実、制約、リマインダーに適しています。

## 注入されるソース

- **プロジェクトメモリ** — `project_memory_write` または `project_memory_add_directive` で書き込まれたキーとなる事実とディレクティブ。`SessionStart` 時に注入され、ディレクティブがアクティブな場合は後続のターンでも選択的に注入されます。
- **セッションリマインダー** — `notepad_write_working` または `notepad_write_priority` で書き込まれた短期メモ。有効期限が切れるか削除されるまで注入されます。
- **Wiki サマリー** — wiki ページが存在する場合に `SessionStart` 時に表示される `.omx/wiki/` のコンパクトな抜粋。フックパスをブロックしないよう、書き込み集中操作（インジェスト、リビルド）は延期されます。
- **モード状態** — アクティブなワークフローモード（ralph、team、ultrawork）が検出された場合、モデルが正しい地点から再開できるよう、現在のフェーズと継続指示が含まれます。

## 例

すべてのコミットにトレーラーを含める必要があるという永続的なディレクティブを注入:

```bash
omx memory add-directive "All git commits must include Co-authored-by: OmX <omx@oh-my-codex.dev>"
```

次のセッション開始時に、このディレクティブが `<system-reminder>` ブロックに自動的に表示されます。

現在の notepad の内容を確認:

```bash
cat .omx/notepad.md
```
