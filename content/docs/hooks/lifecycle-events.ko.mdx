---

OMX 훅은 Codex CLI 라이프사이클 이벤트에 연결됩니다. 각 이벤트는 Codex 세션의 정해진 시점에 발생하며 타입이 지정된 페이로드를 전달합니다. 어떤 이벤트가 언제 발생하는지 이해하면 OMX가 무엇을 할지 예측하고, 자신만의 플러그인 로직을 어디에 추가할지 파악할 수 있습니다.

## 개요

Codex CLI는 외부 훅 핸들러가 구독할 수 있는 고정된 이름의 이벤트 집합을 노출합니다. OMX는 상태 지속성, 키워드 감지, 중단 게이팅, 컨텍스트 주입 등 자체 런타임 관심사를 이 이벤트들에 매핑합니다. 동일한 이벤트 어휘가 `.omx/hooks/*.mjs` 플러그인 파일에도 노출되어, 커스텀 로직이 OMX 내부를 복제하지 않고 올바른 단계에 삽입될 수 있습니다.

## 이벤트

| 이벤트 | 발생 시점 | 주요 OMX 용도 |
|---|---|---|
| `SessionStart` | Codex 세션 초기화 시 | 시작 컨텍스트 복원, 위키 요약 시드, `.omx/`가 gitignore에 포함되었는지 확인 |
| `UserPromptSubmit` | 사용자가 프롬프트를 전송할 때 | 키워드 감지, 스킬 라우팅, deep-interview 입력 잠금 |
| `PreToolUse` | 도구 호출이 실행되기 전 | Bash 안전성 검사, 인라인 `git commit` 형식 가드 |
| `PostToolUse` | 도구 호출이 완료된 후 | 명령어 미발견 안내, 비정상 종료 출력 검토 |
| `Stop` | 모델이 한 턴을 완료할 때 | Ralph/autopilot/ultrawork/ultraqa 계속 실행, 정지 상태 자동 넛지 |
| `PreCompact` | 컨텍스트 창이 한도에 가까워질 때 | 예약됨; 위키 압축은 v2로 연기 |
| `Notification` | 시스템 알림 발생 시 | notify-hook을 통해 Discord/Slack/Telegram으로 알림 라우팅 |

Codex 네이티브 훅 커버리지는 부분적입니다. `PreToolUse`와 `PostToolUse`는 현재 Bash만 인터셉트하며, Bash 이외의 도구 인터셉션은 런타임 경로로 폴백됩니다. 전체 매핑 매트릭스는 [Codex 네이티브 훅](/docs/hooks/codex-native-hooks)을 참고하세요.

## 예제

`post-tool-use` 이벤트에 반응하는 플러그인:

```js
// .omx/hooks/my-plugin.mjs
export async function onHookEvent(event, sdk) {
  if (event.event !== 'post-tool-use') return;
  sdk.log.info('tool finished', event.context);
}
```

이벤트 봉투에는 항상 `schema_version`, `event`, `timestamp`, `source`(`native` 또는 `derived`), `context`가 포함됩니다.
